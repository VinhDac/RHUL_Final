Introduction
The following projects investigate cutting-edge applications of machine learning and AI to video
game development workﬂows, with a particular focus on assisting Quality Assurance (QA) en-
gineers. Students participating in these projects will have opportunities to interact with experi-
enced ML researchers at SIE, and some may be considered for placements or internships. Please
note that both projects include an additional application step involving a short interview with
SIE researchers.
Project 1: VLMs for Video Game QA – Relationships between
Code and Intended Behaviour
Aims
This project aims to explore the capabilities of (Large) Vision Language Models (VLMs) in
understanding the relationship between the behaviour of a game as the player sees it, and the
code that implements this behaviour. Speciﬁcally, it investigates whether VLMs can identify
“regression” bugs, where code or asset changes unintentionally alter game behaviour between
versions.
Background
Quality Assurance (QA) is a vital part of video game development. With recent developments
in AI, there is growing interest in tools that assist QA engineers with the testing process. Video
games present a particularly challenging domain for test automation due to the complexity of
interactions between player inputs and audiovisual output, the numerous subsystems involved
(physics, graphics, AI, networking, etc.), and the subjective nature of “intended behaviour”.
Recent work applying VLMs to game testing has shown promising, though limited, results—
for example, detecting simple bugs such as clipping issues (Taesiri 2025). This project extends
such work by introducing game code as an additional input signal, enabling investigation into
whether VLMs can explain behavioural changes and help identify potential regression bugs.

Early Deliverables
• Literature review on the use of VLMs and related models in video game testing and
automated bug detection.
• Construction of an evaluation dataset containing code/asset diﬀs and gameplay images or
videos, generated from an open-source 3D game engine project, including both intended
and unintended changes.
• Initial evaluation of a chosen VLM’s ability to caption or describe the generated scenes.
Final Deliverables
• Evaluation of whether VLMs can make eﬀective use of code diﬀs to identify behavioural
changes in scenes.
• Evaluation of VLMs’ ability to detect regression bugs with and without code context.
• A report detailing the literature review, design decisions (e.g., prompt engineering), ex-
perimental methodology, and results.
Possible Extensions
• Comparative study of open- and closed-source VLMs applied to this task.

Project 2: Detecting Technical Requirement Violations in Video
Games with (Large) Language Models
Aims
The aim of this project is to develop a machine-learning-based pipeline to assist QA engineers
in ensuring that video games adhere to platform-speciﬁc technical requirements documents.
Background
Gaming platforms impose strict requirements on published titles to ensure a consistent player
experience and compliance with legal obligations. Requirements include correct terminology for
platform features, copyright and trademark compliance, and safeguarding player privacy and
safety. Currently, evaluation against Technical Requirements Checklists (TRCs) is performed
manually, making the process slow and costly.
With recent advances in NLP, particularly LLMs and contextual embedding models, auto-
mated approaches to compliance checking have begun to show promise in other domains such as
legal text analysis (Hassani 2024). This project explores similar approaches for analysing text
extracted from video game screenshots to identify TRC violations.
As with the previous project, this work is conducted in collaboration with SIE, and involves
an additional interview stage for applicants.
Early Deliverables
• Literature review of LLM/RAG approaches to software and video game requirement check-
ing.
• Evaluation of existing embedding models or open-source LLMs on TRC-related tasks.
• A trained or ﬁne-tuned model capable of reliably detecting a subset of TRC violations
from text extracted from real game screenshots.
Final Deliverables
• A pipeline that ingests a screenshot, extracts text, and identiﬁes a subset of TRC violations
across major world languages.
• Robust evaluation of the pipeline, including ablation studies where appropriate and com-
parison to alternative methods.
• A report presenting the literature review, design rationale, and detailed experimental
results.

Possible Extensions
• A model capable of detecting nuanced TRC violations requiring reasoning or broader
contextual understanding, such as those relating to player safety or privacy.
